This is an admin interface for a ticketing system. This was designed with a partner for my Introduction to Databases class at OSU. We worked together to craft a proposal for a mock product that would service a mid-size venue (~1500 seats) to manage all of their data needs. This was accomplished by implementing a relational database after extensive group discussion and numerous peer reviews. This is an SQL MariaDB with multiple entities featuring various kinds of relationships.

Working on this project and the class itself was a wonderful experience. My partner and I were able to communicate effectively, which allowed us to make great progress when we collaborating on a single problem or working asynchronously. It was very satisfying to take the seed of an idea, that was created in a brainstorming session, and grow it into a functioning application.
